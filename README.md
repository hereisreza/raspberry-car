🏎️ سیستم کنترل رباتیک و تله‌متری بلادرنگ (Web-Based RC Car)

یک پلتفرم حرفه‌ای و بهینه‌سازی‌شده برای کنترل ماشین‌های کنترلی (ربات‌های Differential Drive) از طریق مرورگر وب. این سیستم با بهره‌گیری از FastAPI و WebSockets در سمت بک‌اند و یک رابط کاربری فوق‌العاده مدرن و واکنش‌گرا در فرانت‌اند، امکان هدایت ربات با کمترین تاخیر (Low-Latency) و مانیتورینگ دقیق را فراهم می‌کند.

📄 دو فرانت‌اند کاملاً متفاوت — با هم اشتباه نگیرید

این مخزن دو فایل `index.html` دارد که هدف کاملاً متفاوتی دارند:

| فایل | هدف | نیازمند اینترنت؟ | کنترل موتور؟ |
|---|---|---|---|
| `index.html` (ریشه پروژه) | صفحه معرفی عمومی پروژه برای **GitHub Pages** | بله (Tailwind/Chart.js/فونت از CDN) | خیر — هیچ اتصال WebSocket یا فرمان موتوری در این صفحه وجود ندارد |
| `templates/index.html` | **داشبورد واقعی کنترل ربات** که توسط `server.py` روی رسپبری‌پای سرو می‌شود | خیر — کاملاً آفلاین | بله — از طریق WebSocket به مسیر `/ws` |

اگر به دنبال صفحه‌ای هستید که ربات را واقعاً حرکت می‌دهد، همان `templates/index.html` است، نه فایل ریشه پروژه.

✨ ویژگی‌های کلیدی

💻 داشبورد کنترلی آفلاین (Raspberry Pi)

- کاملاً بدون CDN: بدون Tailwind CDN، بدون Google Fonts، بدون jsDelivr/cdnjs و بدون هیچ اسکریپت یا فونت بیرونی. تمام CSS و JS از پوشه `static/` سرو می‌شود.
- جوی‌استیک گاز و فرمان با پیاده‌سازی خالص جاوااسکریپت (Pointer Events)، بدون کتابخانه‌ی خارجی مثل nipplejs.
- گیج سرعت‌سنج داینامیک SVG با فیزیک حرکت (تمایل و چرخش گیج در زمان پیچیدن).
- تله‌متری بلادرنگ (Live Telemetry): CPU، RAM، دمای هسته و قدرت سیگنال Wi-Fi رسپبری‌پای.
- ارتباط دوطرفه WebSocket با نرخ بروزرسانی بالا و بدون نیاز به رفرش صفحه.
- تم تاریک/روشن با فونت‌های سیستمی (بدون دانلود فونت).

⚙️ ایمنی

- **Watchdog ضربان قلب:** اگر مرورگر Heartbeat نفرستد، سرور در کمتر از ۶۰۰ میلی‌ثانیه موتورها را متوقف می‌کند.
- **توقف اضطراری:** دکمه STOP همیشه در دسترس است و بلافاصله فرمان توقف را ارسال می‌کند.
- **بازنشانی خودکار جوی‌استیک:** با قطع اتصال WebSocket، از دست دادن فوکوس پنجره یا مخفی شدن تب مرورگر، مقدار جوی‌استیک صفر شده و فرمان توقف ارسال می‌شود.
- **شتاب‌گیری و ترمز نرم (Slew-rate limiting):** جلوگیری از کشیدن جریان ناگهانی از باتری‌ها.
- **حالت Dry-run:** روی هر سیستمی به‌جز رسپبری‌پای (بدون gpiozero/lgpio)، کنترلر موتور به‌صورت خودکار وارد حالت شبیه‌سازی می‌شود تا توسعه و تست بدون سخت‌افزار ممکن باشد.

⚙️ سخت‌افزار و الکترونیک

- مغز پردازشی: Raspberry Pi Zero 2 W (سیستم‌عامل Debian Bookworm)
- درایور موتور: ماژول پرقدرت BTN7960 (پشتیبانی تا ۴۳ آمپر)
- موتورها: دو عدد موتور گیربکس‌دار صنعتی بُهلر (Buhler) آلمان - ۲۰ ولت
- منبع تغذیه: پک باتری لیتیومی ۵ سلولی (حدود ۱۹ تا ۲۱ ولت) مجهز به مدار محافظ BMS
- ایزولاسیون: استفاده از اپتوکوپلر TLP281 جهت ایزوله کردن مدار فرمان (رزبری‌پای) از مدار قدرت (درایورها).

🛠️ معماری سیستم و نکات مهندسی

۱. محدودیت‌های فرکانس PWM (چالش Bookworm و lgpio)

در سیستم‌عامل دبیان Bookworm، تولید PWM به صورت نرم‌افزاری توسط کتابخانه lgpio انجام می‌شود. فرکانس‌های مجاز سخت‌افزاری/نرم‌افزاری محدود به پله‌های خاصی هستند (۵۰، ۱۰۰، ۵۰۰، ۱۰۰۰، ۴۰۰۰ هرتز). در این پروژه فرکانس از طریق رابط کاربری قابل تنظیم است اما برای پایداری سیستم، حداکثر روی ۴۰۰۰ هرتز محدود شده است.

۲. چالش اپتوکوپلر TLP281

به دلیل کند بودن زمان خاموش شدن (Fall time) در اپتوکوپلرهای TLP281، استفاده از فرکانس‌های بسیار بالا (مثل ۲۰ کیلوهرتز) باعث اعوجاج پالس‌ها و از بین رفتن کنترل در سرعت‌های نزدیک به صفر می‌شود. به همین دلیل فرکانس پیش‌فرض سیستم روی ۵۰۰ هرتز تنظیم شده است تا تعادل کاملی بین "کاهش صدای سوت موتور" و "دقت در سرعت پایین" ایجاد شود.

🚀 راهنمای نصب و راه‌اندازی (روی رسپبری‌پای)

پیش‌نیازها

ابتدا مطمئن شوید که پایتون ۳ و محیط مجازی (venv) روی رسپبری‌پای نصب است:

```
sudo apt update
sudo apt install python3-pip python3-venv
```

نصب وابستگی‌های پایتون

یک محیط مجازی بسازید و کتابخانه‌های مورد نیاز سیستم را نصب کنید:

```
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

یا به‌سادگی اسکریپت آماده را اجرا کنید (venv را خودش می‌سازد و وابستگی‌ها را نصب می‌کند):

```
./run.sh
```

اجرای سرور به صورت دستی

```
uvicorn server:app --host 0.0.0.0 --port 8000
```

سپس در مرورگر خود (موبایل یا لپ‌تاپ، در همان شبکه Wi-Fi رسپبری‌پای) آدرس زیر را باز کنید:

```
http://<RASPBERRY_PI_IP>:8000
```

این آدرس داشبورد آفلاین (`templates/index.html`) را نمایش می‌دهد و نیازی به اینترنت ندارد.

اجرای خودکار در زمان بوت (Systemd Service)

برای اینکه برنامه همیشه روشن بماند، یک سرویس سیستمی بسازید:

```
sudo nano /etc/systemd/system/buhler-drive.service
```

محتویات زیر را در آن قرار دهید (مسیرها را بر اساس پوشه خود اصلاح کنید):

```
[Unit]
Description=Buhler Motor Drive Web Server
After=network.target

[Service]
User=pi
WorkingDirectory=/home/pi/web_drive
ExecStart=/home/pi/web_drive/.venv/bin/uvicorn server:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

سپس سرویس را فعال کنید:

```
sudo systemctl daemon-reload
sudo systemctl enable buhler-drive.service
sudo systemctl start buhler-drive.service
```

🧪 تست روی ویندوز/لینوکس (بدون سخت‌افزار)

روی هر سیستمی به‌جز رسپبری‌پای، `core_motor.py` به‌طور خودکار در حالت Dry-run اجرا می‌شود (بدون نیاز به gpiozero/lgpio واقعی)، بنابراین می‌توانید داشبورد را برای تست رابط کاربری روی لپ‌تاپ خودتان هم اجرا کنید:

```
.\run.ps1
```

سپس آدرس `http://localhost:8000` را باز کنید.

🌐 صفحه GitHub Pages

فایل ریشه‌ی `index.html` یک صفحه‌ی معرفی عمومی است که در GitHub Pages میزبانی می‌شود و صرفاً برای نمایش پروژه (عکس‌ها، شماتیک، ویژگی‌ها) استفاده می‌شود؛ هیچ ارتباطی با کنترل واقعی ربات ندارد و نباید به‌عنوان داشبورد کنترلی استفاده شود.

🎮 راهنمای تنظیمات رابط کاربری

با باز کردن منوی تنظیمات (آیکون چرخ‌دنده)، به پارامترهای زیر دسترسی خواهید داشت:

- حداکثر سرعت: توان نهایی خروجی به موتورها را محدود می‌کند (مناسب برای محیط‌های داخلی).
- فرکانس PWM: تنظیم فرکانس پالس موتور (تغییر زنده بین ۵۰ تا ۴۰۰۰ هرتز).
- نرخ شتاب و ترمز: اینرسی مجازی برای جلوگیری از تیک‌آف و ترمز ناگهانی.
- منحنی حساسیت گاز/فرمان (Expo): افزایش دقت جوی‌استیک در مرکز و سرعت‌های پایین.
- حساسیت فرمان: ضریب ترکیب سرعت موتورهای چپ و راست هنگام دور زدن.
- نقطه مرده (Deadzone): چشم‌پوشی از لرزش‌های ریز انگشت روی جوی‌استیک در مرکز.
- هندسه گردش (مرز گردش درجا، آستانه معکوس، قدرت معکوس چرخ داخلی، حداقل نسبت چرخ داخلی): تنظیم دقیق رفتار چرخش.

⚠️ نکات ایمنی

- همیشه ابتدا ربات را روی سطحی امن و بدون مانع تست کنید.
- قبل از افزایش «حداکثر سرعت»، از عملکرد صحیح دکمه STOP و پایداری اتصال Wi-Fi اطمینان حاصل کنید.
- در صورت مشاهده رفتار غیرمنتظره، فوراً دکمه STOP را بزنید یا تب مرورگر را ببندید (Watchdog به‌صورت خودکار موتورها را متوقف می‌کند).
