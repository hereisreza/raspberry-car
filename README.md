🏎️ سیستم کنترل رباتیک و تله‌متری بلادرنگ (Web-Based RC Car)یک پلتفرم حرفه‌ای و بهینه‌سازی‌شده برای کنترل ماشین‌های کنترلی (ربات‌های Differential Drive) از طریق مرورگر وب. این سیستم با بهره‌گیری از FastAPI و WebSockets در سمت بک‌اند و یک رابط کاربری فوق‌العاده مدرن و واکنش‌گرا در فرانت‌اند، امکان هدایت ربات با کمترین تاخیر (Low-Latency) و مانیتورینگ دقیق را فراهم می‌کند.✨ ویژگی‌های کلیدی💻 نرم‌افزار و رابط کاربری (UI/UX)گیج سرعت‌سنج داینامیک: نمایشگر سرعت SVG با پشتیبانی از فیزیک حرکت (تمایل و چرخش گیج به چپ و راست در زمان پیچیدن ربات).تله‌متری بلادرنگ (Live Telemetry): مانیتورینگ لحظه‌ای وضعیت پردازنده (CPU)، حافظه (RAM)، دمای هسته (Temp) و قدرت سیگنال وای‌فای (WiFi) رسپبری‌پای.ارتباط دوطرفه WebSocket: اجرای فرامین کنترلی با نرخ بروزرسانی ~۳۰ هرتز و دریافت لاگ‌های سیستم بدون نیاز به رفرش صفحه.الگوریتم‌های حرکتی پیشرفته:شتاب‌گیری و ترمز نرم (Ramping): جلوگیری از کشیدن جریان ناگهانی از باتری‌ها (اعمال شده فقط روی محور گاز برای حفظ چابکی فرمان).منحنی حساسیت گاز (Throttle Expo): استفاده از توابع نمایی برای کنترل میلی‌متری و دقیقِ ربات در سرعت‌های پایین.بهینه‌سازی حداکثری فرانت‌اند (High Performance): کش کردن عناصر DOM (DOM Caching) و استفاده از مکانیزم‌های Throttling برای جلوگیری از سرریز پردازشی (Overload) مرورگر و سرور.تم تاریک/روشن (Dark/Light Mode): طراحی مدرن با استفاده از TailwindCSS.⚙️ سخت‌افزار و الکترونیکمغز پردازشی: Raspberry Pi Zero 2 W (سیستم‌عامل Debian Bookworm)درایور موتور: ماژول پرقدرت BTN7960 (پشتیبانی تا ۴۳ آمپر)موتورها: دو عدد موتور گیربکس‌دار صنعتی بُهلر (Buhler) آلمان - ۲۰ ولتمنبع تغذیه: پک باتری لیتیومی ۵ سلولی (حدود ۱۹ تا ۲۱ ولت) مجهز به مدار محافظ BMSایزولاسیون: استفاده از اپتوکوپلر TLP281 جهت ایزوله کردن مدار فرمان (رزبری‌پای) از مدار قدرت (درایورها).🛠️ معماری سیستم و نکات مهندسی۱. محدودیت‌های فرکانس PWM (چالش Bookworm و lgpio)در سیستم‌عامل دبیان Bookworm، تولید PWM به صورت نرم‌افزاری توسط کتابخانه lgpio انجام می‌شود. فرکانس‌های مجاز سخت‌افزاری/نرم‌افزاری محدود به پله‌های خاصی هستند (۵۰، ۱۰۰، ۵۰۰، ۱۰۰۰، ۴۰۰۰ هرتز). در این پروژه فرکانس از طریق رابط کاربری قابل تنظیم است اما برای پایداری سیستم، حداکثر روی ۴۰۰۰ هرتز محدود شده است.۲. چالش اپتوکوپلر TLP281به دلیل کند بودن زمان خاموش شدن (Fall time) در اپتوکوپلرهای TLP281، استفاده از فرکانس‌های بسیار بالا (مثل ۲۰ کیلوهرتز) باعث اعوجاج پالس‌ها و از بین رفتن کنترل در سرعت‌های نزدیک به صفر می‌شود. به همین دلیل فرکانس پیش‌فرض سیستم روی ۵۰۰ هرتز تنظیم شده است تا تعادل کاملی بین "کاهش صدای سوت موتور" و "دقت در سرعت پایین" ایجاد شود.🚀 راهنمای نصب و راه‌اندازیپیش‌نیازهاابتدا مطمئن شوید که پایتون ۳ و محیط مجازی (venv) روی رسپبری‌پای نصب است. سپس پکیج‌های زیر را نصب کنید:sudo apt update
sudo apt install python3-pip python3-venv
نصب وابستگی‌های پایتونیک محیط مجازی بسازید و کتابخانه‌های مورد نیاز سیستم را نصب کنید:python3 -m venv venv
source venv/bin/activate
pip install fastapi uvicorn psutil gpiozero lgpio websockets
اجرای سروربرای اجرای سرور به صورت دستی:uvicorn server:app --host 0.0.0.0 --port 8000
سپس در مرورگر خود (موبایل یا لپ‌تاپ) آدرس آی‌پی رسپبری‌پای را وارد کنید:http://<RASPBERRY_PI_IP>:8000اجرای خودکار در زمان بوت (Systemd Service)برای اینکه برنامه همیشه روشن بماند، یک سرویس سیستمی بسازید:sudo nano /etc/systemd/system/buhler-drive.service
محتویات زیر را در آن قرار دهید (مسیرها را بر اساس پوشه خود اصلاح کنید):[Unit]
Description=Buhler Motor Drive Web Server
After=network.target

[Service]
User=pi
WorkingDirectory=/home/pi/web_drive
ExecStart=/home/pi/web_drive/venv/bin/uvicorn server:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
سپس سرویس را فعال کنید:sudo systemctl daemon-reload
sudo systemctl enable buhler-drive.service
sudo systemctl start buhler-drive.service
